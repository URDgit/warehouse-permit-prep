## PROJECT INTAKE SNAPSHOT
*The five-minute email answers, restated so the engineer confirms scope at a glance. This is also the intake questionnaire — whatever a "yes" reply gives you, these are the blanks to fill:*

- Site address & jurisdiction
- Building: type of construction, sprinklered? (system type/density if known), total building area
- Storage: max storage height, aisle widths, rack vs. floor/pile vs. shelf
- Commodities: what's stored, packaging (cartoned, exposed, encapsulated, on pallets — wood/plastic)
- HPS area: square footage of high-piled storage area(s)
- Any existing permits / prior HPS approvals on the site

## SECTION 1 — AHJ SUBMITTAL CHECKLIST
*What [City] actually requires to accept the application. Sourced from the AHJ's published HPS submittal guideline + permit portal + plan-review bulletins.*

- Governing authority & where it's filed (fire dept vs. building dept; portal name/link)
- Required documents list (application form #, site plan, floor plan showing HPS areas, rack elevations, commodity letter, sprinkler calcs/TI reference, etc.)
- Plan format requirements (scale, sheet size, digital submittal specs)
- Fees & review timeline as published
- **Local quirks:** [the gold — reviewer preferences, common correction letters, local amendments to Ch. 32, unwritten expectations. From bulletins, correction-letter patterns, and portal notes.]
- AHJ contact: plan-review counter phone/email

---

## SECTION 2 — APPLICABLE 2025 CFC CHAPTER 32 SECTIONS
*The specific provisions this project triggers — cited, quoted where short, and mapped to the project. Not a Ch. 32 dump; only what applies, with why.*

Table format: **Section § | Requirement (plain-English summary) | How it applies to this project | Notes/local amendment**

- §3201/3202 scope & definitions as they capture this project
- Commodity classification basis (§3203) → cross-ref Section 3
- Table 3206.2 row(s) that govern (commodity class × HPS area) — the heart of the package
- Storage configuration requirements triggered (aisles, piles, racks — §3206.10, §3207–3209 as applicable)
- Smoke removal / fire department access / hose connections if triggered
- Flag: anything ambiguous → marked "EOR judgment" rather than resolved by us

---

## SECTION 3 — COMMODITY CLASSIFICATION WORKSHEET
*Prepared classification for the engineer to review and adopt/adjust.*

- Inventory table: Product | Packaging | Pallet type | Proposed class (I–IV / High-Hazard / Group A plastics %) | Basis (CFC §3203 / NFPA 13 reference)
- Governing (worst-case) commodity determination & why
- Mixed-commodity notes if applicable
- Explicit line: *Classification proposed for EOR review — final determination is the engineer's.*

---

## SECTION 4 — STORAGE HEIGHT & AREA TRIGGER ANALYSIS
*Which thresholds this project crosses and what each crossing requires.*

- Key numbers: storage height [X ft], HPS area [X sf], commodity class [from §3]
- Trigger table: **Threshold | This project | Crossed? | Consequence (permit scope, protection requirements, doc requirements)**
  - 12-ft HPS trigger · 500 sf / 2,500 sf / 12,000 sf / 500,000 sf area breakpoints as applicable
  - Sprinkler design implications flagged (not designed — flagged for EOR/fire protection engineer)
- One-paragraph plain-English summary: "This project is a [class] / [area] HPS application requiring [scope]."

---

## SECTION 5 — DRAFT PERMIT NARRATIVE
*Formatted to [City]'s conventions, written to be adapted, finalized, and stamped by the EOR.*

- Project description paragraph
- Code analysis summary (pulling §§2–4 into the AHJ's expected narrative structure)
- Commodity & storage configuration statement
- Protection summary (existing/proposed systems, referencing — not performing — the engineering)
- Placeholders marked **[EOR: …]** wherever judgment/design input belongs
- Closing block for engineer's info & stamp
