# Assembly workflow (internal — never ships in a deliverable)

Moved out of `package-skeleton.md` so it can't leak into the client-facing
template (`permitwright-package-template.docx`). This is the internal
production checklist for building each $750 research package.

1. Reply lands → send intake questions (the Project Intake Snapshot fields) same day
2. Pull AHJ docs for the jurisdiction (portal, HPS bulletin, local amendments) → Section 1
3. Run project facts through Ch. 32 → Sections 2 & 4 (engine research = raw material; every citation manually verified against the 2025 CFC before it ships)
4. Build commodity worksheet from intake inventory → Section 3
5. Write narrative last (it summarizes 1–4) → Section 5
6. Sources page as you go, not at the end
7. Self-check before sending:
   - every claim cited
   - every judgment call marked **[EOR: …]**
   - zero "2022" / "7-16" strings anywhere in the document
   - the string "LLC" appears nowhere (until stamped Articles arrive)
   - footer shows the correct project name on every page
   - 48-hour clock met
